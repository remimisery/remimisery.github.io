Stage — Projet phare

Mini-ERP de production "Suivi Commandes" — PHP, Vue.js, MySQL, API REST

Conception et developpement complet d'un plugin WordPress remplacant un fichier Excel partage par un outil de gestion de production centralise pour un groupe e-commerce de 13 boutiques.

Le probleme a resoudre

Avant mon arrivee, le suivi de production reposait sur un fichier Excel partage sur le reseau local de l'atelier, avec tous les problemes que cela implique : acces concurrent impossible (un seul utilisateur a la fois), saisie manuelle de chaque commande depuis WooCommerce, aucune tracabilite des modifications, pas de controle d'acces, risque de perte de donnees, et couverture limitee a un seul site sur treize.

La solution developpee

J'ai concu une application web complete, structuree en architecture API-first. Le back-end PHP expose une API REST avec 10 routes principales (CRUD commandes, mise a jour par poste, avancement d'etat, webhooks). Le front-end Vue.js fonctionne comme une SPA reactive avec des vues dynamiques par poste de travail. La base de donnees MySQL est organisee en six tables personnalisees avec un modele relationnel rigoureux.

Les fonctionnalites cles

  • Import automatique des commandes de 13 sites via l'API WooCommerce, avec un systeme de scan d'IDs puis import batch, et une strategie degradee en trois niveaux en cas de timeout.
  • Workflow de production en six etapes avec machine a etats, gestion de la non-conformite (retrogradation automatique au poste usinage) et avancement automatique.
  • Systeme RBAC avec huit roles et des permissions granulaires : chaque operateur ne voit que ce qui le concerne.
  • Extraction automatique des dimensions et surfaces des meubles par expressions regulieres, et calcul des indicateurs de production.
  • Edition en ligne par double-clic, tri FIFO par anciennete, badges de statut colores, marqueur de version pour le deploiement.

Resultats

Gain de productivite d'environ 80 minutes par jour pour l'atelier. Import de 50 commandes multi-sites passe de plusieurs heures a moins de 10 minutes. 8 exigences du cahier des charges sur 9 realisees, la synchronisation bidirectionnelle etant en cours de developpement.

Projet academique

Base de donnees relationnelle — MySQL et Python

Conception d'une solution complete de gestion de donnees pour une cooperative de sel marin fictive basee sur l'ile de Re : modelisation relationnelle, alimentation automatisee depuis des fichiers CSV, interface graphique Tkinter et requetage SQL avance.

Modelisation

  • MCD sous Looping (methode Merise), traduction en MLD, generation du script SQL de creation avec cles primaires, etrangeres et contraintes d'integrite.
  • Optimisation des requetes SQL complexes pour l'extraction d'indicateurs de performance (ventes annuelles, plus gros clients, analyse par produit).

Automatisation Python

  • Script de detection automatique d'encodage, conversion des formats de date, nettoyage des valeurs manquantes et insertion sans doublons dans MySQL.

Interface graphique

  • Application Tkinter complete : creation de la base, insertion automatique, execution de requetes predefinies ou personnalisees, export CSV, affichage dynamique des tables.

Mon projet prefere de la formation : il m'a donne une vision globale de la gestion de donnees, de la conception a l'exploitation.
