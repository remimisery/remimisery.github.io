📊 Projets réalisés

🧪 Analyse exploratoire de données (Python)

Dans ce projet, j’ai utilisé des jeux de données publics, notamment sur les accidents de la route et la météo, pour me familiariser avec l’analyse de données en Python.

J’ai d’abord nettoyé les données avec Pandas, en gérant les valeurs manquantes et les doublons. J’ai ensuite conçu des visualisations avec Matplotlib, comme des histogrammes pour observer la répartition des accidents selon les heures, ou des heatmaps pour analyser les corrélations entre conditions météo et gravité des accidents.

🔍 Ce projet m’a appris à structurer une analyse, à poser les bonnes questions, et à transformer un fichier brut en informations exploitables. J’ai vraiment pris conscience du pouvoir de la data visualisation pour faire parler les données.

📈 Dashboard RH sur Excel

Ce projet consistait à transformer des données RH brutes (absences, salaires, ancienneté, turnover) en un tableau de bord dynamique.

J’ai utilisé des tableaux croisés dynamiques, des segments pour filtrer par service ou genre, et des formules comme SI, NB.SI, RECHERCHEV, INDEX/EQUIV pour automatiser les calculs.

🔧 Ce projet m’a permis de structurer une interface intuitive pour que n’importe quel utilisateur puisse suivre les indicateurs clés en quelques clics. J’ai compris l’importance d’un bon reporting pour la prise de décision RH, et j’ai pris plaisir à rendre l’outil simple et lisible.

🗃️ Base de données relationnelle (MySQL & Python)

Dans le cadre d’un projet de gestion de données pour une coopérative de sel marin fictive basée sur l’île de Ré, j’ai conçu une solution complète allant de la modélisation des données jusqu’au développement d’une interface graphique utilisateur.

📌 Objectif du projet

Permettre à une coopérative de gérer les apports en sel effectués par les sauniers, les opérations de transformation, les ventes aux clients, ainsi que la tarification annuelle. L’idée était de simuler un système d’information réaliste à partir de données brutes en CSV.

📐 Modélisation de la base de données

  • Réalisation du MCD sous Looping avec la méthode Merise, en tenant compte des contraintes fonctionnelles (produits, clients, sauniers, entrées, sorties...)
  • Traduction en MLD puis génération d’un script SQL de création
  • Ajout de clés primaires, étrangères et de contraintes d'intégrité pour garantir la cohérence des données
  • Utilisation de EasyPHP Devserver 17 et phpMyAdmin pour tester et visualiser la structure

⚙️ Automatisation avec Python

Pour alimenter la base avec les fichiers CSV fournis (récoltes, ventes, produits, clients...), j’ai développé un script Python (selmarin.py) qui :

  • Détecte automatiquement l'encodage de chaque fichier
  • Convertit les formats de date et nettoie les valeurs manquantes
  • Insère les données dans la base MySQL en s’assurant d’éviter les doublons
  • Adapte dynamiquement les colonnes pour respecter les contraintes SQL

🖥️ Interface graphique utilisateur (Tkinter)

J’ai également développé une interface complète avec Tkinter pour :

  • 📂 Créer la base et les tables
  • 📥 Insérer les données automatiquement depuis les fichiers CSV
  • 🔎 Exécuter des requêtes prédéfinies ou personnalisées
  • 🧾 Exporter les résultats de requêtes en CSV
  • 📋 Afficher dynamiquement le contenu des tables

🧠 Requêtes SQL avancées

J’ai conçu et intégré 10 requêtes SQL complexes, comprenant :

  • Requêtes d’analyse avec GROUP BY et HAVING
  • Vues SQL comme Plusgrosclient pour synthétiser les ventes annuelles
  • Manipulations de données : INSERT, UPDATE, DELETE
  • Création et gestion d’utilisateurs SQL avec des permissions spécifiques

📈 Export et historique

  • Export des résultats de requêtes personnalisées au format CSV directement depuis l’interface
  • Affichage d’un historique d’actions et de logs dans l’application

💡 Résultat et apprentissages

Ce projet m’a permis de développer une vision globale de la gestion de données : de la structuration à l’automatisation. J’ai appris à combiner mes compétences en modélisation, en SQL et en Python pour livrer une application fonctionnelle, évolutive, et proche des besoins réels d’une organisation.

✅ C’est à ce jour mon projet préféré, car il m’a permis de mobiliser de nombreuses compétences, de relever des défis techniques concrets, et de produire un résultat complet dont je suis très fier.

📉 Statistique descriptive avec R

Sur RStudio, j’ai travaillé à partir d’un jeu de données réel pour analyser des variables qualitatives et quantitatives.

J’ai produit des boxplots, des nuages de points, et des résumés statistiques (moyenne, médiane, écart-type, etc.).

J’ai aussi exploré la régression linéaire simple, pour comprendre les liens entre deux variables (ex : influence de l’âge sur les revenus).

📊 Ce projet m’a permis de mieux comprendre la logique statistique derrière les chiffres. R m’a aussi montré l’importance de choisir les bons outils visuels pour appuyer une analyse rigoureuse.
